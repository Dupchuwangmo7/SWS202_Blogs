### **1. Introduction to Editing Files**

- After creating files and directories, the next step is to edit and modify them.
- Common text editors in Linux:
    - **Nano** (easy to use, beginner-friendly).
    - **Vim** (powerful, efficient, but requires practice).

---

### **2. Editing with Nano**

- **Nano** is a simple terminal-based text editor.
- To open and edit a file:
    
    ```bash
    nano <filename>
    ```
    
    **Example:**
    
    ```bash
    nano notes.txt
    ```
    
- **Basic Nano Commands:**
    - `CTRL + W` â†’ Search for text.
    - `CTRL + O` â†’ Save file.
    - `CTRL + X` â†’ Exit Nano.
    - `CTRL + K` â†’ Cut text.
    - `CTRL + U` â†’ Paste text.

- **Viewing File Contents in Shell:**
    
    ```bash
    cat notes.txt
    ```

---

### **3. Important System Files**

- **`/etc/passwd`** â†’ Contains user details (username, UID, GID, home directory).
- **`/etc/shadow`** â†’ Stores password hashes (restricted access).
- Misconfigured permissions on these files can create security risks.

---

### **4. Editing with Vim**

- Vim (Vi IMproved) is a powerful and highly configurable text editor.
- It follows the **Unix philosophy**: small specialized programs working together.
- To open Vim:
    
    ```bash
    vim <filename>
    ```
    
- **Vim has different modes:**
    - **Normal Mode** â†’ Default mode for running commands.
    - **Insert Mode** â†’ Allows text input (`i` or `a` to enter insert mode).
    - **Visual Mode** â†’ Used for selecting and manipulating text.
    - **Command Mode** â†’ Used for executing commands (`:` key).
    
- **Basic Vim Commands:**
    - `:q` â†’ Exit Vim.
    - `:wq` â†’ Save and exit.
    - `:q!` â†’ Quit without saving.
    - `i` â†’ Enter insert mode.
    - `ESC` â†’ Return to normal mode.

---

### **5. Learning Vim with VimTutor**

- Vim provides an interactive tutorial for beginners.
- To start the tutorial:
    
    ```bash
    vimtutor
    ```

- **Estimated time to complete**: 25-30 minutes.

---

### **Conclusion**

- Nano is easier for quick edits, while Vim is more efficient for advanced users.
- Understanding permissions and system files is crucial for security.
- Learning Vim can significantly improve text editing speed in Linux.
